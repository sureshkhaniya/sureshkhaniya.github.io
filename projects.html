Airlines Ticketing & Reservation System

Project Type: Academic Desktop Application

Technical Stack: C#, WPF (XAML), .NET, Microsoft Access (ADO.NET)

This project is a desktop-based airline ticketing and reservation system designed to simulate real-world airline operations with a focus on aircraft configuration, seat allocation, and passenger management.

The system allows users to manage flights, register passengers, and assign seats dynamically based on aircraft capacity. Business rules are enforced to prevent overbooking and to ensure that seat assignments remain consistent across all reservations.

A relational database backend is used to persist passenger, flight, and reservation data. Data access responsibilities are separated from the user interface through dedicated service and manager classes, reinforcing clean architectural boundaries.

The user interface was developed using WPF and XAML, emphasizing structured layouts, input validation, and event-driven interaction. This project highlights how desktop applications coordinate real-time user input with backend data storage.

Core Competencies: Object-Oriented Design, Desktop Application Development, Database Integration, Seat Allocation Logic, Event-Driven Programming

Future Enhancements: Introduce pricing tiers by seat class, implement authentication roles (agent vs administrator), and migrate to a scalable SQL-based backend.

Banking System Application

Project Type: Academic Software Development Project

Technical Stack: Java

This project involved the design and implementation of a Java-based banking system that simulates core financial operations such as account creation, deposits, withdrawals, and balance inquiries.

The application enforces strict business rules to ensure valid transactions and system consistency. Defensive programming techniques were applied to handle invalid inputs and prevent logical errors.

A strong emphasis was placed on object-oriented principles, including encapsulation, responsibility separation, and maintainable class design. The system models realistic banking workflows while remaining extensible for future features.

Core Competencies: Business Logic Design, Input Validation, Defensive Programming

Future Enhancements: Add persistent storage, authentication mechanisms, and concurrency controls to more closely resemble real-world banking systems.

Word Association Learning Activity (Node.js / Express / EJS)

Project Type: Upper-Division Academic Application (Multi-Module)

Technical Stack: Node.js, Express.js, EJS, JavaScript, HTML/CSS

The Word Association Learning Activity is an iterative Node.js application developed across multiple modules to reinforce user interface design, server-side routing, and application structure using Express and EJS.

The application includes a user registration workflow where users submit personal and demographic information through dynamically rendered EJS forms. Additional user fields were identified through documentation review and integrated using appropriate form controls.

Express routers handle both GET and POST requests, and browser developer tools were used to inspect network traffic and verify correct form submission behavior. The application also introduces role-based workflows by separating administrative views from standard user interactions.

While the current implementation uses simulated data, the architecture is intentionally designed to support database integration in later modules.

Core Competencies: Express Routing (GET/POST), EJS Templating, Form Handling, UI Design, Role-Based Application Structure

Future Enhancements: Integrate persistent storage, implement real statistical queries, and expand authentication and authorization features.
